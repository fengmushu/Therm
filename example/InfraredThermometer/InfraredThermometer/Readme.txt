功能描述
================================================================================
本额温枪样例为开发过程版本，旨在提供基本功能模块单元的操作及架构，为应用开发者提供快速原型构建，
将更多的精力用在业务开发上及系统调试上。

================================================================================
测试环境
================================================================================
辅助工具:
---------------------

配套硬件参考:
---------------------
“Hm-Meter-源文件.rar”及“Hm-Meter-PDF BOM.rar”
硬件参考资料可向销售窗口获取

================================================================================
操作步骤
================================================================================
1) 程序编译后下载到配套方案板；
2) 复位或初次上电LCD全显闪烁两次，保持全显；
3) 参考软件流程（操作）进行验证；

================================================================================
功能说明
================================================================================
1）打开工程编译并运行。
2）可根据不同的具体硬件IO功能修改"board_stkhc32l13x.h"板级接口文件，或自行定义功能。
3）数据采集部分使用内部ADC连续16次采样取均值的方式采样外部经过OPA调理过的信号。
4) 显示部分提供了配套原理图“Hm-Meter-源文件.rar”及“Hm-Meter-PDF BOM.rar” 所使用LCD的数据结构及操作，同时包括HC32L13X_STK的LCD参考。
5) 数据处理算法参考“nna.h"及样例。
6) 按键及其他基础功能模块较为简单，通常为基本的IO操作，本应用提供了基本的初始化操作。

================================================================================
注意
================================================================================
-->>本样例实现了额温枪方案的基本方法及架构，基于华大MCU HC32L136进行构建，算法已基于现有硬件配套传感器进行验证；
-->>仅可为应用开发者提供快速原型参考，具体的端口、业务功能、数据处理应以实际采纳的硬件为准，进行程序修改及调试；
-->>如果硬件发生更改，请对应修改软件及配置;
-->>如果传感器发生更改，请更新“inf_config”中的V-T表及NTC表;
-->>LCD使用如有不同，需要根据手册修改编码及端口配置;
-->>本应用示例对于硬件及软件逻辑功能不做任何限制，主要用于参考原型，便于对进行自己产品的差异化开发;
